## 3. Cross-Container File Transfer `🚂 RAILWAY`

### 3.1 Problem
Laravel dan Python service jalan di container terpisah. Python tidak bisa akses file di filesystem Laravel (`/var/www/html/storage/...`).

### 3.2 Solution: Base64 Transfer
**File:** `app/Services/PythonServiceClient.php`
**Perubahan:**
```php
// Baca file & encode base64
$fileContent = base64_encode(file_get_contents($filePath));
$fileName = basename($filePath);

$payload = [
    'file_path' => $filePath,
    'file_name' => $fileName,        // NEW
    'file_content' => $fileContent,  // NEW (base64)
    // ... rest of payload
];
```

**File:** `python-service/main.py`
**Perubahan:**
```python
class ProcessRequest(BaseModel):
    file_name: Optional[str] = None       # NEW
    file_content: Optional[str] = None    # NEW (base64)

def _process_report_background(request):
    if request.file_content:
        file_bytes = base64.b64decode(request.file_content)
        file_path = storage_dir / file_name
        file_path.write_bytes(file_bytes)

    # BUGFIX: pakai file_path lokal, bukan request.file_path
    clean_result = cleaner.run(str(file_path))  # Sebelumnya: request.file_path
```

**Rollback impact:** Jika rollback, lokal dev masih bisa jalan karena `if request.file_content:` — tanpa base64, Python tetap pakai `request.file_path` (path lokal).

## 5. Callback Handler Fix `🔧 GENERAL`

### 5.1 Foreign Key Violation
**File:** `app/Http/Controllers/Api/ReportCallbackController.php`
**Error:** `SQLSTATE[23000]: Integrity constraint violation: 1452 ... ai_provider_id = 0`
**Root Cause:** `$provider?->id ?? 0` — ketika provider tidak ditemukan di DB, fallback ke `0` yang tidak ada di tabel `ai_providers`.
**Fix:**
```php
// SEBELUM:
$provider = AIProvider::where('name', 'LIKE', "%{$providerName}%")->first();
'ai_provider_id' => $provider?->id ?? 0,  // ❌ FK violation

// SESUDAH:
$provider = AIProvider::where('slug', $providerName)->first()
         ?? AIProvider::where('name', 'LIKE', "%{$providerName}%")->first();
if (!$provider) {
    Log::warning("Provider '{$providerName}' not found");
    continue;  // ✅ Skip instead of crash
}
'ai_provider_id' => $provider->id,
```

## 6. AI Provider isReady() Fix `🚂 RAILWAY`

### 6.1 Problem
Panel admin menampilkan "No API key" untuk semua provider karena `isReady()` cek `env('GEMINI_API_KEY')` di Laravel container — tapi API keys hanya ada di Python service container.

### 6.2 Fix
**File:** `app/Models/AIProvider.php`
```php
// SEBELUM:
public function isReady(): bool {
    return $this->is_enabled && $this->getApiKey() !== null;
    // getApiKey() cek env() → null di production → "No API key"
}

// SESUDAH:
public function isReady(): bool {
    return $this->is_enabled && !empty($this->api_key_env);
    // Hanya cek apakah env var NAME dikonfigurasi, bukan nilainya
}

public function getApiKey(): ?string {
    $key = env($this->api_key_env);
    if (!empty($key)) return $key;
    // Fallback: di production, keys di Python service
    if ($this->is_enabled && !empty($this->api_key_env)) {
        return '[configured-in-python-service]';
    }
    return null;
}
```
**Rollback:** Aman di lokal — jika env var ada (dev), return key asli. Jika tidak, return placeholder.

## 13. Markdown Rendering di Narasi AI `🎨 UI`

### 13.1 Problem
Narasi AI menghasilkan teks dengan markdown formatting (`**bold**`, `*italic*`, `# heading`, dll), tapi komponen `NarrativeBox.vue` menampilkan semua sebagai plain text.

**Root cause:** Vue template menggunakan `{{ formattedNarrative }}` — text interpolation yang *escape* semua HTML/special chars.

### 13.2 Fix
**File:** `resources/js/Components/NarrativeBox.vue`

1. **Custom `renderMarkdown()` parser** (tanpa library tambahan):
```js
const renderMarkdown = (text) => {
    // Escape HTML entities
    // Parse headings: # ## ###
    // Parse bold: **text**
    // Parse italic: *text*
    // Parse bold+italic: ***text***
    // Parse strikethrough: ~~text~~
    // Parse lists: - item, 1. item
    // Parse inline code: `code`
    // Parse horizontal rule: ---
    // Wrap paragraphs: double newline → <p>
}
```

2. **Template:** `{{ formattedNarrative }}` → `v-html="formattedNarrative"`
```html
<!-- SEBELUM -->
<div class="whitespace-pre-line">{{ formattedNarrative }}</div>

<!-- SESUDAH -->
<div class="narrative-content" v-html="formattedNarrative"></div>
```

3. **Print window** juga diupdate dengan CSS styles untuk heading, bold, italic, list, code, dll.

### 13.3 Formatting yang didukung
| Markdown | HTML | Tampilan |
|----------|------|----------|
| `**teks**` | `<strong>` | **teks** |
| `*teks*` | `<em>` | *teks* |

## 14. Chart Image Base64 Transfer `🚂 RAILWAY`

### 14.1 Problem
Setelah processing, chart gambar menampilkan icon broken image (404 Not Found). Console menunjukkan:
```
Failed to load resource: the server responded with a status of 404 ()
  distribusi_numeric.png
  distribusi_kategori.png
  tren_waktu.png
```

**Root cause:** Sama seperti file upload — Python service dan Laravel jalan di container terpisah. Chart disimpan di Python container (`/app/storage/charts/`), tapi Laravel cari di **storage sendiri** (`/var/www/html/storage/app/public/charts/`) → 404.

### 14.2 Fix — 3 File Diubah

**File 1:** `python-service/chart_generator.py`
```python
# SEBELUM: Simpan ke Laravel path (cross-container = tidak bisa)
laravel_base = os.environ.get('LARAVEL_BASE_PATH', '')
chart_folder = Path(laravel_base) / 'storage/app/public/charts'

# SESUDAH: Simpan lokal di Python container
chart_folder = Path(__file__).parent / 'storage' / 'charts' / str(report_id)

# Return type berubah:
# List[str] → List[Dict[str, str]]
# Sekarang return {path, data (base64), filename}
import base64
with open(chart_file, 'rb') as f:
    b64_data = base64.b64encode(f.read()).decode('utf-8')
results.append({'path': relative, 'data': b64_data, 'filename': name})
```

**File 2:** `python-service/main.py`
```python
# SEBELUM:
'chart_paths': chart_paths,  # List[str]

# SESUDAH:
chart_path_list = [c['path'] for c in chart_paths]
chart_image_list = [{'path': c['path'], 'data': c['data'], 'filename': c['filename']}
                     for c in chart_paths if c.get('data')]
callback_data = {
    'chart_paths': chart_path_list,      # List[str] (relative paths untuk DB)
    'chart_images': chart_image_list,    # List[Dict] (base64 data untuk transfer)
}
```

**File 3:** `app/Http/Controllers/Api/ReportCallbackController.php`
```php
// Validasi tambahan:
'chart_images' => 'nullable|array',
'chart_images.*.path' => 'nullable|string',
'chart_images.*.data' => 'nullable|string',
'chart_images.*.filename' => 'nullable|string',

// Decode & simpan:
if (!empty($validated['chart_images'])) {
    foreach ($validated['chart_images'] as $chartImage) {
        $decoded = base64_decode($chartImage['data']);
        Storage::disk('public')->put($chartImage['path'], $decoded);
        // Tersimpan di: storage/app/public/charts/{id}/filename.png
        // Accessible di: /storage/charts/{id}/filename.png (via storage:link)
    }
}
```
