# DataNarasi - Project Context (2026-04-28)

## 🎯 Project Overview
**DataNarasi** is a full-stack web application that automates data analysis. Users upload CSV/Excel files, and the system performs automated data cleansing, statistical analysis, chart generation, and ultimately uses an AI provider to write a business narrative/insight in Bahasa Indonesia.

## 🏗️ Architecture & Tech Stack
- **Backend (Laravel 13):** Handles user authentication (Breeze), file uploads, database management (MySQL), and asynchronous background jobs (Queue).
- **Frontend (Vue 3 + Inertia.js + TailwindCSS):** Provides a reactive SPA interface for users to upload data and view reports. Interactive charts via Chart.js + vue-chartjs.
- **Microservice (Python 3.11 + FastAPI):** Handles heavy data processing. It uses Pandas for data cleansing/analysis, Matplotlib/Seaborn for charts, and various OpenAI-compatible SDKs for AI narrative generation.

## 🤖 Multi-AI Fallback System
The core feature of this project is a resilient multi-AI fallback chain. If one AI provider fails (e.g., rate limit, timeout), the system automatically falls back to the next one in the chain.

**Current Fallback Order (Configurable via `AI_PROVIDER_ORDER` in `.env`):**
1. Gemini 3 Flash Preview
2. Kimi (Moonshot)
3. GLM (Zhipu AI)
4. NVIDIA NIM (Llama 3.1 8B Instruct)
5. MiniMax (M2.5)
6. Claude (Anthropic - Paid)

**Admin Panel:** Providers can be managed at `/admin/ai-providers` — toggle on/off, edit API keys, change model IDs, adjust max tokens/timeout, and drag-to-reorder priority. Requires `is_admin = true`.

## 🔄 How the Processing Pipeline Works
1. User uploads a file in Laravel with analysis type, tone, and optional title.
2. Laravel queries `ai_providers` table (sorted by priority) and sends provider order + file to Python FastAPI (`/process`).
3. Python FastAPI immediately returns `202 Accepted` and runs the pipeline in a `BackgroundTask` to avoid cURL timeouts.
4. Pipeline steps: Data Cleansing -> Statistical Analysis -> Chart Generation -> AI Narrative Generation.
5. AI provider order follows the admin panel priority (dynamic per-request, not hardcoded).
6. Once done, Python sends an HTTP POST callback back to Laravel (`/api/v1/reports/{id}/callback`).
7. Callback is signed with HMAC-SHA256 and includes retry mechanism (3 attempts, exponential backoff).
8. Laravel validates HMAC signature, updates the report status, saves AI usage logs, and the UI updates automatically.

## 🔒 Security Architecture
- **Config Cache Safety:** All `env()` calls centralized in `config/python-service.php` — safe for `php artisan config:cache`.
- **HMAC Callback Signing:** Python→Laravel callbacks signed with HMAC-SHA256 using shared secret. Laravel verifies payload integrity.
- **Admin Role-Based Access:** `is_admin` column on `users` table + `IsAdmin` middleware guards all `/admin/*` routes.
- **CORS Restriction:** Python FastAPI CORS reads from `ALLOWED_ORIGINS` env (not wildcard `*`).
- **API Key Security:** Admin panel stores API keys directly in `.env` file — never exposed in UI or database.

## 🚀 Running the Project

### Quick Start (Recommended)
```bash
composer run dev
```
Runs all 4 services concurrently: Laravel (`:8000`), Queue Worker, Vite HMR (`:5173`), Python FastAPI (`:8001`).

### Environment Variables (Key ones)
```
PYTHON_SERVICE_URL=http://localhost:8001
PYTHON_SERVICE_SECRET=rahasia-internal-token-ganti-ini
PYTHON_SERVICE_TIMEOUT=10
ALLOWED_ORIGINS=http://localhost:8000
AI_PROVIDER_ORDER=gemini,kimi,glm,nvidia,minimax,claude
```
